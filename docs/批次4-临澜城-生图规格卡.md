# 批次 4 · 临澜城分层重做 · 生图规格卡

> 复刻批次 1 v2 范式（绿幕底 / 比例与入库尺寸分离 / 三查 / 风格链一处定义 / 落位契约脚本实测）。
> **本批是最贵的一张（10 楼 + 2 散件 + 1 张 1664×928 大地貌），排在批次 1-3 之后最后做。**
> 用法：每张卡的**提示词直接复制**进生图工具，按编号顺序做（①是风格基准、②是建筑族基准）。
> 生成完放进 `art/raw/`，按卡上的**文件名**命名，跟我说"放好了"——量化/抠绿/校验/集成/碰撞/测试/截图证据全是我的活。

## 共同硬规则（每张都要）

1. **背景一律纯绿幕**：物件卡（②-⑬）的背景必须是**一整片纯绿色**（提示词已写好 solid pure green #00FF00）。你的工具出不了透明 PNG，绿幕是这个项目一直在用的老办法（我负责抠绿）。**别要求透明底，别用棋盘格底**。
2. **工具里选比例，分辨率不用管**：每张卡写了"工具里选什么比例"。生成出来是 1024 还是 2048 都行，**最终尺寸我来缩**（两步法：先采回原生艺术分辨率再整数放大，禁止非整数一步缩——临澜城 v1 的"颗粒烂泥"就是这么来的）。
3. **30 秒像素粗筛**：出图后放大到 400% 看边缘——合格 = 一格一格的硬台阶；不合格 = 模糊渐变、毛玻璃边。**拿不准就直接给我，我跑行程统计脚本验**，不合格我会指出重 roll 哪张。
4. **物件卡（②-⑬）统一三查**：① 背景是一整片纯绿、绿色没混进物体本体 ② 画面里只有这一个物件——没有地面、没有影子 ③ 400% 看边缘是硬台阶。
5. 整体冷蓝灰调；**全批次禁琥珀禁暖棕**（琥珀是凤翎专属的全画面唯一暖色）；**病态黄绿也禁用**（临澜城距灰域 300km，是人类与 AI 协同的日常市井，没有异常侵染）。建筑/设备灯光一律青绿 #6fd3c4。临澜城和监测带的区别是"有人住"：可以旧、可以有生活痕迹，但依然冷。

> **🔩 引擎真相（⑫⑬ 集成必读，照旧数据走必出事故）**：bgImage 模式只渲染 `p.movable && p.sprite` 的物件（index.html ≈L2111），且 sprite 一律 `setDisplaySize(size×16)` 拉伸到占格像素（≈L2113）——**不存在"按原生尺寸锚定底格"的渲染路径**。所以：
> 1. 两件必须改 **movable:true**（否则 movable:false 一帧都不渲染，交互点隐形——正是本卡要防的事故）；
> 2. **size 必须 = sprite 像素 ÷16**（charge_post 16×32 → size[1,2]；notice_screen 48×32 → size[3,2]），否则 1×1 占格会把 sprite 压扁成 16×16；
> 3. pos 相应上移让底格不变（新坐标见落位契约表⑫⑬行，已 python 实测验证）。

5. ⚠️ 参考图只学色调——如果生成结果里**长出了地面**或**整体发绿**，说明参考权重太高，调低或去掉参考重 roll
